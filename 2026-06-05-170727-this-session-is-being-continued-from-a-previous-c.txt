  ~/.claude.json 是 Claude Code CLI 自己维护的全局配置文件，每次你在新目录里跑
  claude，CLI 就会把这个目录写进 projects map（用来存各项目的 per-project
  配置，比如自定义指令、允许的工具等）。所以截图里看到的
  /Users/xmly、/Users/xmly/.cc-panes 这些，都是曾经跑过 claude 
  的目录，自动收录的。

  ---
  我们的实现：
  扫描 ~/.claude/projects/*/*.jsonl transcript 文件：

  ~/.claude/projects/
    <encoded-cwd>/        ← 工作目录路径的 "/" 换成 "-"
      <session-uuid>.jsonl

  Discoverer.Scan() 遍历这个目录树，解析每个 .jsonl 提取 WorkingDir、摘要等，以
  Origin=external 导入 session.Manager。

  ---
  对比总结：

  Column 1: 数据源
  ccmate: ~/.claude.json["projects"]
  我们: ~/.claude/projects/*/*.jsonl
  ────────────────────────────────────────
  Column 1: 来源类型
  ccmate: Claude CLI 的全局 per-project 配置
  我们: Session transcript 文件
  ────────────────────────────────────────
  Column 1: 内容
  ccmate: 目录路径 + 项目配置（allowed tools 等）
  我们: 完整会话记录（可提取摘要、标题、最后活跃时间）
  ────────────────────────────────────────
  Column 1: 目的
  ccmate: 展示/编辑项目配置
  我们: 列举实际会话并生成 AI 摘要
  ────────────────────────────────────────
  Column 1: 数量
  ccmate: 所有跑过 claude 的目录
  我们: 只有存在 transcript 的会话

  两者互补，不是同一层面的东西。ccmate 适合配置管理，我们的更适合会话管理。如果
  想让我们的项目列表也能覆盖"曾经用过但没有飞书会话"的目录，可以把
  ~/.claude.json["projects"] 作为额外数据源叠加进 projectsForUser。
